# نکسا متریکا — حافظه‌ی پروژه (Project Memory)

پلتفرم آنالیز فوتبال هوش‌مصنوعی‌محور، دوزبانه فارسی(RTL)/انگلیسی(LTR).
فایل اصلی: `Nexa Metrica.dc.html` (یک Design Component واحد).

## هویت بصری (ثابت)
- تم تیره خاکستری-زغالی؛ توکن‌ها به‌صورت CSS vars روی root: `--bg:#0d0f12 --card:#16191c --ac:#a3e635 (لایم، برند/داده) --ai:#38bdf8 (سایان، AI) --warn:#f59e0b --dng:#ef4444 --good:#4ade80`.
- فونت Vazirmatn. اعداد با `faN()` به فارسی.
- ساختار: سایدبار (راست در RTL) با ۶ گروه + نوار بالا (جستجو، کلید نقش، زبان، اعلان، دستیار) + پنل کشویی دستیار AI.
- روی هر خروجی AI: نشان «تولید AI» + درصد اطمینان + دکمه‌ی «چرا؟».

## ساختار ناوبری
سایدبار **آبشاری/تاشو**: هر گروه با کلیک هدر باز/بسته می‌شود (گروه فعال همیشه باز). state.navCollapsed.

## صفحه‌ی `leaguedb` — بازیکنان و تیم‌های لیگ
سایدبار: آیتم «بازیکنان و تیم‌های لیگ» **والد آبشاری** با دو زیرمنو «تیم من» و «لیگ»؛ کلیک روی خودِ والد → نمای **مدیریت لیگ‌ها** (state.ldbView: manage/myteam/league).
✅ نمای «تیم من»: جدول بازیکنان خودی · ✅ نمای «لیگ»: ریل لیگ‌ها/تیم‌ها + جدول · ✅ نمای «مدیریت»: کاربر **نام لیگ** را وارد می‌کند و AI/ML کل لیگ (تیم‌ها+بازیکنان) را استخراج می‌کند (state.customLeagues، addLeagueByName، شبیه‌سازی fetch). لیگ‌های فعال با شمار تیم/بازیکن و حذف.
✅ **سورت پوزیشن فوتبالی** (دروازه‌بان→مهاجم via posRank، نه الفبایی) روی همه‌ی ستون‌ها · فیلتر پوزیشن/تیم · جستجو · Excel.

## نقش‌ها (state.role, ROLES)
آنالیزور ارشد ✅ / آنالیزور ✅ / مربی ✅ / بازیکن ✅ / مدیر باشگاه ✅ / **سوپر ادمین ✅**.
هر نقش: هویت + منوی فیلترشده + صفحه‌ی خانه. کلید «مشاهده به‌عنوان نقش» در نوار بالا (با بستن بیرونی).
